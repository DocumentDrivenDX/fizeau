## Intent Summary

- **Vision**: Deliver an embeddable Go agent runtime with replayable sessions, an auditable tool surface, reusable routing policy, and interoperable analytics.
- **Requirements**: The repo now ships JSONL replay, usage aggregation, symlink-resolved file operations, and enforced CI quality-gate wiring. Remaining repo-wide execution gaps are narrowed to conversation continuity, full telemetry contract conformance, one streaming timing correctness fix, and one stale streaming design doc.
- **Features / Stories**: FEAT-001 and FEAT-002 are implemented and tested. FEAT-006 `usage` is implemented end to end. FEAT-005 has meaningful runtime scaffolding but is not fully contract-conformant yet. PRD P1 session continuity is still missing.

- **Technical Design**: SD-001 and SD-002 now match the shipped usage/cost direction. SD-004 still says usage arrives only on the final streaming delta, which no longer matches the provider implementations or `consumeStream`.
- **Test Plans**: `go test ./...`, `CGO_ENABLED=1 go test -race ./...`, `go vet ./...`, and `golangci-lint run` all passed on 2026-04-09. `gosec ./...` reported active findings. `govulncheck ./...` was blocked because the installed binary was built with Go 1.25 while this module requires Go 1.26.
- **Implementation Plans**: The ready/open queue is now concentrated in five follow-ups: continuity (`agent-d1ed4f25`), telemetry completion (`agent-3027ae82` and `agent-dba02a2e`), streaming timing correctness (`hx-fd626e74`), streaming-doc refresh (`hx-364abdde`), and the `gosec` backlog (`hx-4c5b2879`).
