    <title>execute-bead: add wall-clock deadline alongside idle timeout for agent harness</title>
    <description>
## Root cause RC2 from ddx-0a651925&#xA;&#xA;The `DefaultTimeoutMS = 7200000` (2h) in `cli/internal/agent/types.go:363` is advertised as a hard cap but implemented as an *idle timeout*:&#xA;&#xA;- `cli/internal/agent/executor.go:154-178` resets `time.NewTimer(idleTimeout)` on every stdout/stderr byte.&#xA;- `cli/internal/agent/agent_runner.go:264-285` resets on every non-compaction agentlib event.&#xA;&#xA;A provider that emits any heartbeat, retry line, or periodic event defeats the timer indefinitely — enabling the 33–142h hangs observed on agent projects.&#xA;&#xA;Only the claude streaming path (`cli/internal/agent/claude_stream.go:373-384`) uses a true wall-clock deadline via `time.After(timeout)`. That path is why bundle `20260415T015231-c2ffa9ba` fired at exactly 2h 0m 0.098s — and nothing else does.&#xA;&#xA;## Scope&#xA;&#xA;1. Add an absolute wall-clock deadline in addition to (not replacing) the idle timeout.&#xA;2. Default the wall-clock bound to something operators can trust — e.g. 3x `DefaultTimeoutMS` or a new `DefaultWallClockMS`.&#xA;3. Apply in: `OSExecutor.ExecuteInDir` (subprocess path) and `Runner.RunAgent` (embedded agent path).&#xA;4. Reflect the wall-clock error in result.json as a distinct `error: &#34;wall-clock deadline exceeded after Xh&#34;` so it is not confused with idle timeout.&#xA;&#xA;## Non-goals&#xA;&#xA;- Lowering or removing the existing idle timeout.&#xA;- Per-provider tuning (can be follow-up).
    </description>
    <acceptance>
- [ ] New default wall-clock deadline (e.g. 3h) applied to both `OSExecutor.ExecuteInDir` and `Runner.RunAgent`&#xA;- [ ] Wall-clock timeout fires even when stdout/stderr/agent events are flowing (unit test: fake provider emits 1 event/sec for &gt; deadline, expect cancel at deadline ± 1s)&#xA;- [ ] Idle timeout behaviour preserved (unit test: no activity for idle timeout → fire)&#xA;- [ ] Result.json distinguishes `timeout: wall-clock` from `timeout: idle`&#xA;- [ ] Documented in types.go (DefaultWallClockMS constant with comment)
