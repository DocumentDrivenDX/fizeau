    <title>claude harness: write per-iteration trace into execution bundle&#39;s embedded/ dir</title>
    <description>
The embedded ddx-agent harness honors the per-run SessionLogDir override (agent_runner.go:136-138: `logDir := opts.SessionLogDir; if empty fall back to r.Config.SessionLogDir`) so that execute-bead runs drop their JSONL trace inside the bundle at .ddx/executions/&lt;attempt&gt;/embedded/agent-*.jsonl.&#xA;&#xA;The claude harness does NOT honor that override. claude_stream.go:327 reads only r.Config.SessionLogDir, so every claude execute-bead run writes its trace into the runner-wide default (`.ddx/agent-logs/`) and the bundle&#39;s embedded/ dir stays empty. execute_bead.go:446 already passes SessionLogDir: embeddedStateDir through RunOptions; it just isn&#39;t used by the claude path.&#xA;&#xA;Empirical evidence from axon&#39;s 2026-04-15 ADR-018 session: 2 runs with `harness: agent` produced 34-51 MB traces in embedded/; 23 runs with `harness: claude` left embedded/ empty while `.ddx/agent-logs/` grew to 162 MB.&#xA;&#xA;Fix: plumb opts.SessionLogDir into runClaudeStreaming (either accept it as a parameter or read it off resolvedOpts) and prefer it over r.Config.SessionLogDir when non-empty. Matches the embedded harness behavior exactly.&#xA;&#xA;Parity matters: users want per-run forensic traces attached to the bundle regardless of which harness ran the bead.
    </description>
