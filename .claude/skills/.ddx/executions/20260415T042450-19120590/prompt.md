    <title>refactor(graphql): resolver shape cleanup — stable-key cursors, constructor-enforced state, snapshot-vs-domain decision</title>
    <description>
Follow-up from Stage 2.4 opus review: the resolver pattern established in Stage 2.2 (node/projects) and extended in Stage 2.4 (documents/docGraph) has 3 load-bearing shape issues that should be fixed before Stage 2.5+ compounds them across 7 more resolver beads.&#xA;&#xA;1. Stable-key cursors. Current: resolver_node.go defines encodeCursor(i int) / decodeCursor(s string) that base64-wrap &#34;cursor:&lt;slice-index&gt;&#34;. Slice-index cursors break when the underlying collection reorders between pages. Fine for projects (rarely mutate) but broken for beads, documents, commits, sessions, workers (mutated or resorted per query). Replace with ID-based or (sort-key, ID)-based stable cursors. New file: cli/internal/server/graphql/cursors.go with encodeStableCursor(id string) / decodeStableCursor(s string).&#xA;&#xA;2. Constructor-enforced non-nil state. Current: every resolver has &#34;if r.State == nil { return }&#34; or &#34;if r.WorkingDir == empty { return }&#34; early returns. As Stage 2 grows this becomes 50+ copy-pasted guards. Replace with a NewResolver(state StateProvider, workingDir string) *Resolver constructor that rejects nil/empty and returns an error, drop the runtime checks.&#xA;&#xA;3. Snapshot struct vs direct domain types. Current inconsistency: resolver_node.go uses ProjectSnapshot / NodeStateSnapshot (adapter structs) while resolver_documents.go uses docgraph.Document directly. Pick ONE approach and migrate the other. Recommendation: direct domain types (simpler, less boilerplate). Migrate resolver_node.go to drop ProjectSnapshot.&#xA;&#xA;Not in scope: the prefix-registry for node(id) dispatch (concern 5 from the Stage 2.3 note) — keep that for when bead/doc/worker prefixes are added. Not in scope: splitting StateProvider into sub-interfaces (concern 1) — decide that lazily when the interface actually grows.&#xA;&#xA;File: cli/internal/server/graphql/{cursors.go,resolver.go,resolver_node.go,resolver_documents.go,state_graphql.go}
    </description>
    <acceptance>
(1) New cli/internal/server/graphql/cursors.go with encodeStableCursor and decodeStableCursor helpers that encode/decode a stable string key (not slice index). (2) resolver_node.go and resolver_documents.go both use the new helpers; old encodeCursor(i int) is deleted. (3) NewResolver constructor exists, rejects nil State and empty WorkingDir with a clear error. All &#34;if r.State == nil&#34; / &#34;if r.WorkingDir == empty&#34; guards removed from individual resolvers. (4) ProjectSnapshot removed — resolver_node.go returns docgraph or server domain types directly, matching the pattern in resolver_documents.go. (5) All existing integration tests (TestGraphQLNodeInfo/TestGraphQLProjects/TestGraphQLNodeByProjectID/TestGraphQLDocuments/TestGraphQLDocGraph) still pass. (6) go build ./... clean.
