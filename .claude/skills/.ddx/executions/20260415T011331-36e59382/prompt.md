    <description>

The parent execute-loop never observes task_succeeded and hangs forever.&#xA;&#xA;## Observed&#xA;&#xA;Running &#39;ddx agent execute-loop --once --local --harness agent --model qwen/qwen3-coder-next&#39; against axon-43ed769b (axon auth-core-types bead):&#xA;&#xA;1. qwen3-coder-next ran the task correctly, producing commit 91d4bdb with 636 LOC of types + 109 green tests + clippy clean. After ~20 minutes the session log was 33 MB (8461+ events, 249 messages) and stuck in:&#xA;&#xA;    {&#34;type&#34;:&#34;compaction.start&#34;,...}&#xA;    {&#34;type&#34;:&#34;compaction.end&#34;,...,&#34;data&#34;:{&#34;messages_after&#34;:249,&#34;messages_before&#34;:249,&#34;no_compaction&#34;:true,&#34;success&#34;:false}}&#xA;    {&#34;type&#34;:&#34;compaction.start&#34;,...}&#xA;    {&#34;type&#34;:&#34;compaction.end&#34;,...,&#34;data&#34;:{&#34;messages_after&#34;:249,&#34;messages_before&#34;:249,&#34;no_compaction&#34;:true,&#34;success&#34;:false}}&#xA;    ...&#xA;&#xA;3. A1 (previous bead) ran the same harness/model and terminated correctly in ~14 min with identical-looking commit shape. The difference was bead size: A2&#39;s output was a single 633-line Rust file, vs A1&#39;s 350-line SQL + 290-line test file. A2 didn&#39;t exceed what A1 did — so the context-limit explanation doesn&#39;t fully account for it.&#xA;&#xA;## Expected&#xA;&#xA;- After the agent commits and has nothing more to do, it should emit task_succeeded and terminate&#xA;- Compaction should either succeed (messages_after &lt; messages_before) or bail out cleanly, not retry indefinitely&#xA;- If the context genuinely cannot be compacted, the agent should report a structured failure and preserve the commits, not spin&#xA;&#xA;## Scope&#xA;&#xA;- Detect mid-turn compaction returning success=false repeatedly and bail with a structured &#39;compaction_stuck&#39; status&#xA;- Bound the total number of compaction attempts in a single turn&#xA;- Ensure task termination happens immediately after a successful commit + no pending tool calls&#xA;- Add a watchdog at the execute-loop level: kill the child if no commit+progress event is observed within a budget&#xA;&#xA;## Evidence&#xA;&#xA;- Session log: /Users/erik/Projects/axon/.ddx/executions/20260415T005652-ebe7a16a/embedded/agent-1776214612561376672.jsonl (33 MB)&#xA;- Rescued commit: 91d4bdb on axon master&#xA;- Parent process: PID 4141811 (killed)&#xA;&#xA;## Acceptance&#xA;&#xA;- [ ] Compaction retry loop is bounded and fails structurally&#xA;- [ ] Post-commit termination path is deterministic&#xA;- [ ] Watchdog detects and terminates stuck sessions with a preserved-commit outcome&#xA;- [ ] Regression test reproduces the stuck state (seeded session with large no-op tool calls)
