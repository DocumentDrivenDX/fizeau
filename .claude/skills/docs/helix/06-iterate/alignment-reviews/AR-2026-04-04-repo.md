- **Vision**: DDx is shared infrastructure for document-driven development; documents, not code, are the primary managed artifact.

- **Technical Design**: `docs/helix/02-design/plan-2026-04-04.md` is the current implementation plan for the remaining gaps.
- **Test Plans**: There is no separate `docs/helix/03-test/` layer; the current executable coverage is the Go test suite.
- **Implementation Plans**: The plan doc is in place and matches the current split between completed core surfaces and the two active follow-up areas.

## Implementation Map

- **Topology**: `cli/cmd` owns the Cobra command tree; `cli/internal/docgraph`, `cli/internal/bead`, `cli/internal/agent`, `cli/internal/server`, `cli/internal/mcp`, and `cli/internal/config` provide the runtime engines; `website/content` owns the public docs site; `cli/internal/server/frontend` is the embedded SPA.
- **Entry Points**: `cli/main.go`, `cli/cmd/server.go`, `cli/cmd/doc.go`, `cli/cmd/bead.go`, `cli/cmd/agent_cmd.go`, `cli/internal/server/server.go`, `website/content/_index.md`.
- **Test Surfaces**: `go test ./...` in `cli` passes; key suites include `cli/internal/docgraph`, `cli/internal/bead`, `cli/internal/server`, `cli/internal/agent`, `cli/internal/mcp`, and the CLI contract/acceptance tests under `cli/cmd`.
- **Unplanned Areas**: The embedded SPA does not yet expose document editing, persona browsing, or UI search across document content. The registry/package-install command family is absent from `cli/cmd`.
