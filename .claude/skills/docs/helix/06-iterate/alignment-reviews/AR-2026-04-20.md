## Implementation Map

- **Topology**: Monorepo producing `ddx` CLI (`cli/`), `ddx-server` (embedded in same binary, in-process HTTP + MCP + GraphQL + SvelteKit UI), and `ddx.github.io` website (`website/` Hugo+Hextra).
- **Entry points**: `cli/main.go` → Cobra CommandFactory in `cli/cmd/`. Server bootstrap in `cli/internal/server/server.go`. Frontend entry at `cli/internal/server/frontend/src/routes/+layout.svelte`.
- **Public interfaces**: `cli/internal/bead/store.go` (bead store), `cli/internal/agent/runner.go` + `service_run.go` (agent dispatch via upstream `agentlib.Service`), `cli/internal/exec/store.go` (execution collections), `cli/internal/server/graphql/schema.graphql` (GraphQL surface), `cli/internal/server/server.go` (HTTP + MCP).
- **Test surfaces**: Go acceptance/integration/unit tests in `cli/cmd/*_test.go` and `cli/internal/*/*_test.go`; Playwright e2e in `cli/internal/server/frontend/e2e/*.spec.ts` (10 spec files, ~2.1 kLOC); Hugo microsite e2e in `website/e2e/`.
- **Feature flags / config switches**: `.ddx/config.yaml` (`agent.routing.profile_priority`, harness selection, limits); `DDX_NODE_NAME`; `XDG_DATA_HOME` for server state; ts-net via ADR-006.
- **Build and deploy surfaces**: `Makefile` + `cli/Makefile` (Go build, frontend embed via `//go:embed all:frontend/build`); `.github/workflows/ci.yml` (lint, test, build matrix, frontend e2e); `.github/workflows/security.yml` (govulncheck, bun audit, gitleaks, gosec, CodeQL).
- **Major unplanned code paths**: Legacy React files (`src/App.tsx`, `src/pages/*.tsx`) remain in tree despite SD-022 Stage 3 cleanup bead.
- **Dead / orphaned implementations**: `fakeExecuteBeadGit` and `fakeAgentRunner` persist in agent execute-bead tests; concerns.md flags them as migration debt.
