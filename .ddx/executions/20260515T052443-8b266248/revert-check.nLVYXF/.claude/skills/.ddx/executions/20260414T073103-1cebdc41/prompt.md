    <title>Add ddx agent workers command — native live worker status with current bead</title>
    <description>
There is no CLI command to see which workers are running and which bead each is executing. The WorkerManager in cli/internal/server/workers.go already tracks this (WorkerRecord.CurrentBead at line 51, WorkerRecord.State, WorkerRecord.Harness, WorkerRecord.Model, WorkerRecord.Attempts/Successes/Failures) but it is only accessible through the server API. Local --local workers are not tracked by the server at all.&#xA;&#xA;This bead adds ddx agent workers as a top-level CLI subcommand. It must not depend on worktrunk (wt) or any external tool.&#xA;&#xA;## Required changes&#xA;&#xA;### 1. New command: ddx agent workers (cli/cmd/agent_cmd.go)&#xA;Register alongside the existing ddx agent log command. Flags:&#xA;  --json         emit raw JSON array&#xA;  --watch        re-render every 2s (like watch -n2) until Ctrl-C&#xA;  --project      filter to a specific project root (default: CWD project)&#xA;&#xA;### 2. Data sources (two paths)&#xA;A. Server-submitted workers: GET http://127.0.0.1:8080/api/agent/workers — deserialize []WorkerRecord from the server. Already implemented in cli/internal/server/workers.go List() / HTTP handler.&#xA;B. Local --local workers: the server does not track these. Detect them by:&#xA;   1. Reading .ddx/workers/*/status.json files in the project root (server writes these even for local workers when they self-register — confirm in workers.go; if not, fall back to scanning active execute-bead worktrees)&#xA;   2. Scanning .ddx/.execute-bead-wt-* directories (from ExecuteBeadWtPrefix constant in cli/internal/agent/execute_bead.go line 403) and reading the manifest.json inside each to extract bead_id, attempt_id, harness, model, started_at&#xA;&#xA;### 3. Local worker detection via manifest scan&#xA;For each directory matching .ddx/.execute-bead-wt-*:&#xA;  - Read .ddx/executions/{attemptID}/manifest.json (path encoded in worktree name suffix)&#xA;  - Check if worktree is still active (git worktree list --porcelain)&#xA;  - Read execute-loop-heartbeat-at from bead.Extra (requires ddx-7651506a) if available&#xA;  - Derive synthetic WorkerRecord from manifest fields&#xA;&#xA;### 4. Table output (default)&#xA;Columns: STATE | BEAD | TITLE (truncated 40 chars) | HARNESS | MODEL | ELAPSED | ATTEMPTS&#xA;- STATE: running (green), exited (dim), failed (red)&#xA;- ELAPSED: human-readable (e.g., 4m32s)&#xA;- If current bead title unavailable (local worker pre-manifest), show bead ID&#xA;&#xA;### 5. Merge and deduplicate&#xA;When both server and manifest sources are available, deduplicate by worker_id or attempt_id. Server record takes precedence over manifest-derived record for the same worker.&#xA;&#xA;## Key files&#xA;- cli/cmd/agent_cmd.go — add newAgentWorkersCommand(), register in newAgentCommand()&#xA;- cli/internal/server/workers.go — WorkerRecord struct (lines 31-54), List API&#xA;- cli/internal/agent/execute_bead.go — ExecuteBeadWtPrefix constant (line 403), manifest shape (lines 726-771)&#xA;- cli/internal/agent/types.go — SessionEntry Correlation map (bead_id field)
    </description>
    <acceptance>
ddx agent workers prints a table with STATE, BEAD ID, TITLE, HARNESS, MODEL, ELAPSED, ATTEMPTS columns. Server-submitted workers show state from WorkerManager. Local --local workers are detected via manifest scan of .ddx/.execute-bead-wt-* directories. --json flag emits raw JSON. --watch re-renders every 2s. A worker executing bead axon-abc123 shows that bead ID and its title (truncated). An idle system (no active workers) prints &#39;no active workers&#39; rather than an empty table. No dependency on worktrunk or any external binary beyond git.
