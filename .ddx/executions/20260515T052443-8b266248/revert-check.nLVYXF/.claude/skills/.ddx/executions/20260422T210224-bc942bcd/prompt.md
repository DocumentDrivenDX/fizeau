    <title>Bead detail header: long bead ids overlap action buttons (shrink-0 prevents truncation)</title>
    <description>
## Observed&#xA;&#xA;On the bead detail panel, a long bead id overflows and visually overlaps the Claim/Unclaim/Edit/Delete/Close buttons on the right. Reporter example: `.execute-bead-wt-ddx-0a651925-20260418T043148-1346e8a3-526efaf1` (~60 chars).&#xA;&#xA;## Root cause&#xA;&#xA;`cli/internal/server/frontend/src/lib/components/BeadDetail.svelte:211-221` —&#xA;&#xA;```svelte&#xA;&lt;div class=&#34;flex shrink-0 items-center justify-between border-b ... px-6 py-4&#34;&gt;&#xA;  &lt;div class=&#34;flex min-w-0 items-center gap-3&#34;&gt;&#xA;    &lt;span class=&#34;shrink-0 font-mono text-xs ...&#34;&gt;{bead.id}&lt;/span&gt;       ← shrink-0 prevents truncation&#xA;    &lt;span class=&#34;shrink-0 font-medium ...&#34;&gt;{bead.status}&lt;/span&gt;&#xA;    {#if bead.owner}&#xA;      &lt;span class=&#34;truncate text-xs ...&#34;&gt;@ {bead.owner}&lt;/span&gt;&#xA;    {/if}&#xA;  &lt;/div&gt;&#xA;  &lt;div class=&#34;ml-3 flex shrink-0 items-center gap-2&#34;&gt;&#xA;    [Claim/Unclaim] [Edit] [Delete] [Close]&#xA;  &lt;/div&gt;&#xA;&lt;/div&gt;&#xA;```&#xA;&#xA;The bead-id span has `shrink-0`, so it never truncates regardless of length. The outer panel is `max-w-xl` (576px). A 60-char monospace id occupies roughly 400-500px on its own; combined with the status pill, owner chip, and the action-button group (also `shrink-0`), it pushes past the available width. Flex layout collapses by overlapping rather than wrapping, so the id draws on top of the buttons.&#xA;&#xA;## Proposed direction&#xA;&#xA;1. **Replace `shrink-0` on the bead-id span with `truncate min-w-0`.** The parent `&lt;div class=&#34;flex min-w-0 items-center gap-3&#34;&gt;` already establishes a shrinkable container.&#xA;2. **Add `title={bead.id}` so hovering shows the full value** (important — truncating monospace ids from the right loses the distinguishing suffix).&#xA;3. **Add a copy-to-clipboard icon button** next to the id so operators can grab the full value with one click when the id is truncated. Small icon button, no extra visual weight.&#xA;4. **Sanity check:** the reporter&#39;s example starts with `.` and has the shape of an execute-bead worktree name. Worth briefly checking whether something is producing such a value as a bead id (worktree name leaking into bead creation?) — but that&#39;s a separate investigation. The UI still needs to degrade gracefully regardless.&#xA;&#xA;## Out of scope&#xA;&#xA;- Changing bead id generation or enforcing max length on ids.&#xA;- Redesign of the detail-panel header layout.
    </description>
    <acceptance>
**User story:** As a developer opening a bead with a long id, the id truncates cleanly without overlapping any action button, the full id is discoverable on hover and copyable with one click, and the header buttons remain fully clickable.&#xA;&#xA;**Acceptance criteria:**&#xA;&#xA;1. The bead-id span in `BeadDetail.svelte` uses `truncate min-w-0` (or equivalent) instead of `shrink-0`. The parent flex container keeps `min-w-0`.&#xA;2. The span has `title={bead.id}` so hovering reveals the full value.&#xA;3. A copy-to-clipboard icon button sits immediately after the id. Clicking copies the full id to the clipboard and surfaces a subtle confirmation (existing toast / inline check mark — whichever matches project conventions).&#xA;4. Action buttons (Claim/Unclaim, Edit, Delete, Close) remain fully visible and clickable regardless of id length. Status pill and owner chip remain visible; they may wrap below the id on narrow viewports but never overlap the action group.&#xA;5. Playwright e2e:&#xA;   - Seeds a bead with a 60-char id (e.g. `.execute-bead-wt-ddx-0a651925-20260418T043148-1346e8a3-526efaf1`) and opens its detail panel.&#xA;   - Asserts the id element has a truncation class and that its rendered width is ≤ the parent container width minus the action-button group width.&#xA;   - Asserts the Delete button is clickable (`.click()` does not throw) and that its bounding rect does not intersect the id span&#39;s bounding rect.&#xA;   - Asserts hovering the id shows a tooltip (or reveals `title` attribute) with the full value, and clicking the copy button writes the full id to the clipboard.&#xA;6. Unit/visual: at `max-w-xl` width the header renders without horizontal scroll and without clipped action buttons for ids of length 8, 32, 60, and 120.
    </acceptance>
