    <title>exec: ddx agent run --output text — plain result extraction</title>
    <description>
ddx agent run currently prints the agent&#39;s full session (JSONL events) to stdout regardless of flags. To extract just the final assistant text message, callers have to parse JSONL and walk the session log. This breaks trivial automation like &#39;review=$(ddx agent run ... --prompt @/tmp/r.md)&#39;.&#xA;&#xA;## Scope&#xA;- --output text: emit only the final assistant text message to stdout (no tool calls, no JSONL)&#xA;- --output json-result: emit only the result-type JSON object (session metadata + final text)&#xA;- --output session-jsonl: current behavior; keep as non-default opt-in&#xA;- Default switches to --output text&#xA;&#xA;## Context&#xA;Observed during axon ADR-018 loop: both review runs spilled 90-200KB of session JSONL to stdout, requiring python3 JSON-walking to extract the ~2KB opus verdict. --output text would have made the review invocation one line instead of three.
    </description>
    <acceptance>
- [ ] --output flag accepts text | json-result | session-jsonl&#xA;- [ ] Default is text&#xA;- [ ] Integration test covers all three modes&#xA;- [ ] No behavior change for session-jsonl mode
    </acceptance>
