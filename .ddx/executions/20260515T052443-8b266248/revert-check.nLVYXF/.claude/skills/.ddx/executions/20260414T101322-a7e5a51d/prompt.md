    <title>Extend ddx agent log with --bead filter and per-bead attempt history</title>
    <description>
ddx agent log is fully implemented (cli/cmd/agent_cmd.go lines 745-807) and reads .ddx/agent-logs/sessions.jsonl. Each SessionEntry has a Correlation map (cli/internal/agent/types.go) that includes bead_id when the session was created by execute-bead. However there is no way to filter sessions by bead — you must know the session ID in advance or scroll through all recent sessions.&#xA;&#xA;This bead adds --bead &lt;id&gt; filtering and a per-bead attempt summary view to the existing command.&#xA;&#xA;## Required changes&#xA;&#xA;### 1. --bead flag on ddx agent log (cli/cmd/agent_cmd.go)&#xA;When --bead &lt;bead-id&gt; is provided:&#xA;  - Scan all SessionEntry records in sessions.jsonl&#xA;  - Filter to entries where Correlation[&#34;bead_id&#34;] == bead-id&#xA;  - Sort ascending by Timestamp (attempt order)&#xA;  - Display in attempt-history table format (see below)&#xA;&#xA;No change to the default behavior (no --bead = show recent N sessions as today).&#xA;&#xA;### 2. Attempt history table columns&#xA;ATTEMPT | STARTED | DURATION | HARNESS | MODEL | OUTCOME | TOKENS | COST | SESSION&#xA;- ATTEMPT: 1-based index in chronological order&#xA;- OUTCOME: merged / preserved / no-changes / error (from ExecuteBeadResult.Outcome via Correlation or result.json)&#xA;- SESSION: truncated session ID (8 chars) for use with ddx agent log &lt;session-id&gt;&#xA;- COST: formatted as /bin/zsh.0042 or &#39;local&#39; if cost_usd == 0&#xA;&#xA;### 3. Outcome resolution&#xA;SessionEntry does not directly store outcome — it stores exit_code and error. The full outcome is in .ddx/executions/{attemptID}/result.json (ExecuteBeadResult struct). When rendering the attempt table:&#xA;  - Use Correlation[&#34;attempt_id&#34;] to find .ddx/executions/{attempt_id}/result.json&#xA;  - Read ExecuteBeadResult.Outcome from that file&#xA;  - Fall back to exit_code == 0 ? &#39;success&#39; : &#39;error&#39; if result.json is absent&#xA;&#xA;### 4. Summary line&#xA;After the table, print a one-line summary:&#xA;  &#39;&lt;bead-id&gt;: N attempts, M merged, K preserved, J errors. Last attempt: &lt;elapsed&gt; ago.&#39;&#xA;&#xA;### 5. --json passthrough&#xA;When --bead and --json are both set, emit the filtered SessionEntry array as JSON with an added outcome field resolved from result.json.&#xA;&#xA;## Key files&#xA;- cli/cmd/agent_cmd.go — newAgentLogCommand() lines 745-807, add --bead flag and filter path&#xA;- cli/internal/agent/types.go — SessionEntry struct (lines 94-122), Correlation map field&#xA;- cli/internal/agent/execute_bead.go — ExecuteBeadResult struct (lines 30-64), artifact paths convention&#xA;- .ddx/agent-logs/sessions.jsonl — runtime data source
    </description>
    <acceptance>
ddx agent log --bead axon-abc123 shows all execution attempts for that bead in chronological order with ATTEMPT, STARTED, DURATION, HARNESS, MODEL, OUTCOME, TOKENS, COST, SESSION columns. OUTCOME is resolved from result.json when available, falls back to exit_code. Summary line printed after table. --json emits filtered SessionEntry array with outcome field. Bead with no sessions prints &#39;no sessions found for bead &lt;id&gt;&#39;. Existing ddx agent log behavior (no --bead flag) is unchanged. ddx agent log &lt;session-id&gt; single-session detail view is unchanged.
    </acceptance>
