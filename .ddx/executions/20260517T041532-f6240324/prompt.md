    <title>openrouter: drop from candidate pool when credential or credit is missing</title>
    <description>
PROBLEM&#xA;Fizeau&#39;s auto-route includes openrouter as an eligible candidate even when no API key is configured AND/OR the account has zero credit balance. Dispatches that pick openrouter then fail at HTTP-call time with 401 (no auth header) or insufficient-credit errors. Beads burn retry cycles, sticky leases get assigned to a dead provider, and operators must debug a transport error to discover that the actual problem is a missing credential or empty wallet.&#xA;&#xA;This is the same pattern Fizeau already handles correctly for local LMStudio/MLX endpoints: when a local endpoint fails its aliveness probe, it surfaces as `filter_reason: &#34;endpoint_unreachable&#34;` in routing_decision and is excluded from scoring. Openrouter needs an analogous health gate.&#xA;&#xA;EVIDENCE FROM DOWNSTREAM (DDx, commit 09881f3a5)&#xA;While debugging fizeau-80d4bf41 (auto-route candidate enumeration), I observed:&#xA;1. With `--model sonnet-4.6` pinned, Fizeau routed to `fiz/openrouter/anthropic/claude-sonnet-4.6` with score 196.0 (auto-route&#39;s first choice once the model name was known to it).&#xA;2. The dispatch then immediately failed: `agent: provider error: openai: POST &#34;https://openrouter.ai/api/v1/chat/completions&#34;: 401 Unauthorized {&#34;message&#34;:&#34;Missing Authentication header&#34;,&#34;code&#34;:401}`.&#xA;3. No upstream filter on the openrouter candidate. It appeared in the candidate list as `eligible: true`, was selected, and the credential failure surfaced only at HTTP call time inside the harness.&#xA;&#xA;Symmetric symptom is expected when credentials exist but credit is exhausted — openrouter returns a 402/insufficient-credit response. Without proactive credit checks, every dispatch through openrouter will burn a retry to discover this.&#xA;&#xA;PROPOSED FIX&#xA;Add an openrouter provider-health hook analogous to the LMStudio aliveness probe. The hook should:&#xA;&#xA;1. **Credential check** (cheap, run on every routing pass or cache for a short TTL like 60s):&#xA;   - If the openrouter API key environment variable / config is empty or unset → mark openrouter `eligible: false` with `filter_reason: &#34;credential_missing&#34;`. Do not even hit the network.&#xA;   - If the configured key is malformed (wrong prefix, wrong length) → same.&#xA;&#xA;2. **Credit check** (more expensive, run periodically — recommend 5-15 minute TTL with stale-allowed-during-grace):&#xA;   - Call openrouter&#39;s `GET /api/v1/credits` (or whatever the current endpoint is) to fetch remaining credit balance.&#xA;   - If balance is below a configurable threshold (default: $0.50) → mark openrouter `eligible: false` with `filter_reason: &#34;credit_exhausted&#34;`, include current balance in the decision evidence.&#xA;   - If the credit-check call itself fails with 401 → mark `credential_invalid` (different from missing).&#xA;   - If the credit-check call fails with transport error → mark `provider_unreachable` (transient; fail-open with stale-signal-penalty bump rather than hard-excluding, to avoid flapping).&#xA;&#xA;3. **Surface in routing_decision**: candidate row appears with `eligible: false`, `filter_reason` set to one of the values above, and `health_freshness_at` / `quota_freshness_at` populated from the cached probe. This mirrors how `bragi-club-3090` already appears with `endpoint_unreachable` and how `claude-haiku-4.5` appears with `power_missing`.&#xA;&#xA;4. **Cache placement**: probe state should live in the same place as the existing harness aliveness/quota state so the route picker reads it without doing per-call network I/O. `service_routing.go` already has `health_freshness_at` and `quota_freshness_at` fields plumbed through — wire openrouter into that.&#xA;&#xA;NON-SCOPE&#xA;- Multi-account credential rotation / failover within openrouter.&#xA;- Per-model credit-cost forecasting.&#xA;- The DDx-side change is none — this is purely Fizeau-side. DDx passes `policy=default` through unchanged per CONTRACT-003.&#xA;&#xA;CROSS-REF&#xA;Filed in tandem with fizeau-80d4bf41 (auto-route candidate enumeration collapses multi-tier subscription harnesses to DefaultModel only). The two bugs together explain why DDx&#39;s `policy=default` lands on opus instead of sonnet: enumeration drops sonnet from claude, model-pin fallback routes to openrouter, openrouter has no creds.
    </description>
