    <title>Make route-time local liveness nonblocking for autorouting</title>
    <description>
A dead local OpenAI-compatible provider (example: grendel rapid-mlx at http://grendel:8000/v1) can still delay or poison unpinned route selection before the routing engine chooses a healthy subscription harness. The current shared service path calls buildRoutingInputsWithCatalog(..., RefreshIfStale), which synchronously calls refreshLocalHealthForRouting and synchronous model snapshot refresh before scoring. This affects ResolveRoute directly, Execute via resolveExecuteRouteWithEngine, route-status, and override telemetry&#39;s stripped ResolveRoute call.&#xA;&#xA;Required contract: local/fixed provider liveness and model discovery must be cached signals on the route hot path. Stale or missing local-provider facts may request a coordinated asynchronous refresh, but they must not block unpinned autorouting or explicit non-local harness selection. Provider contact failures must be recorded as route evidence/cooldown when observed, but failure to contact a local provider must not prevent selection of another eligible harness/provider. If every candidate is unavailable, routing must still fail with a typed no-viable-provider style error rather than hanging or aborting the process.&#xA;&#xA;In-scope files:&#xA;- service_routing.go&#xA;- service_aliveness.go&#xA;- service_execute.go&#xA;- service_override.go only if context propagation requires it&#xA;- internal/modelsnapshot/types.go and internal/modelregistry/assemble.go comments if the freshness contract text is stale&#xA;- service_aliveness_test.go and service_routing_test.go for regression coverage&#xA;- docs/helix/01-frame/features/FEAT-004-model-routing.md and/or docs/helix/02-design/adr/ADR-012-per-source-on-disk-cache.md to align the spec with the nonblocking route hot path&#xA;- CHANGELOG.md&#xA;&#xA;Out-of-scope:&#xA;- DDx retry policy changes&#xA;- provider config schema changes&#xA;- changing explicit provider pins to silently fall back to a different provider&#xA;- requiring a DDx daemon for correctness
    </description>
