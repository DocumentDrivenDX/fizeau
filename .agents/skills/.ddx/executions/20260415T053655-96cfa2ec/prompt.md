    <title>exec: post-merge review agent on-by-default in execute-loop</title>
    <description>
execute-loop currently closes a bead on merge. There is no automated post-merge review step validating the commit against the bead&#39;s AC. Real-world practice requires a second-opinion agent (typically a higher-capability tier) to produce APPROVE / REQUEST_CHANGES.&#xA;&#xA;## Principle&#xA;&#xA;Review is on-by-default with zero flags. The tool picks a tier &gt;= the implementation tier (defaulting to smart). The user overrides only in exceptional cases.&#xA;&#xA;## Default flow&#xA;&#xA;- Implementation agent runs at current tier (see ddx-dc231144 tier-based escalation)&#xA;- On successful merge, tool automatically renders a review prompt via &#39;ddx bead review&#39; (ddx-fdf031c9)&#xA;- Review agent runs at max(impl_tier + 1, smart)&#xA;- Review result is structured JSON: { verdict: APPROVE | REQUEST_CHANGES | BLOCK, findings: [...] }&#xA;- APPROVE → close bead, attach review evidence&#xA;- REQUEST_CHANGES → reopen bead via &#39;ddx bead reopen&#39; (ddx-44084b3e), append review body, let the tier-escalation policy pick the next attempt&#xA;- BLOCK → stop escalating, flag for human&#xA;&#xA;## Rare overrides&#xA;&#xA;- --no-review (skip review entirely — e.g., doc-only beads in a tight loop)&#xA;- --review-harness / --review-model (explicit reviewer override for debug)&#xA;- Per-bead opt-out via a &#39;review:skip&#39; label on the bead&#xA;&#xA;## Dependencies&#xA;&#xA;- ddx-fdf031c9 (ddx bead review command) provides the review prompt template&#xA;- ddx-44084b3e (ddx bead reopen) provides the atomic reopen primitive&#xA;- ddx-518b2690 (plain --output text) makes extracting the review verdict trivial&#xA;- ddx-5bac3c6b (provider health probe) feeds the reviewer tier selection&#xA;&#xA;## Context&#xA;&#xA;Observed during axon ADR-018 execute-loop:&#xA;- qwen3-coder-next merged A1 with real AC gaps (missing round-trip tests, zero Postgres coverage, scope creep)&#xA;- Without review, the bead would have closed silently and the gaps would surface only at epic rollup — much later and much more expensive to fix&#xA;- Review took 41s + a few minutes of reading. Cheap relative to the real cost of shipping broken code downstream.
    </description>
