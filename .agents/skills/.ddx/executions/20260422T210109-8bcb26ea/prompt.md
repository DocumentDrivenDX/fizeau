    <title>Sessions &#39;Total cost&#39; is ambiguous: split into cash paid, subscription-equivalent, and local compute</title>
    <description>
## Observed&#xA;&#xA;The Sessions page ($ sums `session.cost` into one &#34;Total Cost&#34; card, `cli/internal/server/frontend/src/routes/.../sessions/+page.svelte:21,28,91`). The underlying `cost_usd` field is fed from three very different billing realities and the UI treats them as one:&#xA;&#xA;1. **Cash paid (pay-per-token).** API providers (OpenRouter, direct OpenAI, direct Anthropic API). `cost_usd` = actual billed amount.&#xA;2. **Subscription-equivalent.** Claude Code, Codex — flat-rate subscriptions. The CLI/harness emits `total_cost_usd` as the **dollar-equivalent** for tokens consumed under the subscription (what you would have paid pay-per-token). You paid a flat fee up front; this is a value-extracted number, not a cash-out number.&#xA;3. **Local compute.** Embedded `agent` harness, ollama/vllm endpoints. Today emits cost 0 because no one is billing; real cost is electricity + hardware amortization, not currently modeled.&#xA;&#xA;A user reading &#34;Total Cost: $X&#34; does not know which of those X is.&#xA;&#xA;## What&#39;s needed&#xA;&#xA;Three independent rollups, each explicitly labeled, each computable without ambiguity:&#xA;&#xA;1. **Cash** — sum of `cost_usd` across sessions whose billing mode is `paid`.&#xA;2. **Subscription-equivalent** — sum across sessions whose billing mode is `subscription`. Copy: &#34;Dollar-equivalent for tokens consumed via Claude Code / Codex subscription. You did not pay this in cash; it&#39;s the pay-per-token rate you would have been charged.&#34;&#xA;3. **Local** — either a count of sessions (if no rate is configured) or a computed dollar estimate if `local_cost_per_1k_tokens` is set in config. Default: count-only, to avoid inventing numbers.&#xA;&#xA;## Backing data model&#xA;&#xA;Session records (the sharded index from `ddx-2ceb02fa`) need a new field `billingMode: &#34;paid&#34; | &#34;subscription&#34; | &#34;local&#34;`. Derive at write time:&#xA;&#xA;- `paid`: harness is an API-endpoint provider (openrouter, openai-compat with an API key, anthropic API, …). Probably any harness with `Surface == &#34;openai-compat&#34;` or a known public-api provider prefix.&#xA;- `subscription`: harness is `claude` (Claude Code) or `codex` or `gemini-cli` when using a Google account. Identify by harness name + surface.&#xA;- `local`: harness is `agent` (embedded) OR endpoint is localhost / private IP / configured-as-local in provider config.&#xA;&#xA;Classification lives in one place — a small `billingModeFor(harness, surface, baseURL)` function — so the logic isn&#39;t duplicated between writer and reader.&#xA;&#xA;## Scope&#xA;&#xA;### Part 1 — Data model&#xA;&#xA;- Extend `SessionEntry` / the session index row with `billingMode string` (enum validated on write).&#xA;- Add `billingModeFor(...)` helper + unit tests for each harness/surface combination.&#xA;- Back-fill: `ddx agent log reindex` (from `ddx-2ceb02fa`) computes `billingMode` when re-shard-writing legacy sessions.&#xA;&#xA;### Part 2 — Backend rollup&#xA;&#xA;- Modify the sessions GraphQL query (or add a sibling `sessionsCostSummary(projectId, since?)`) to return `{ cashUsd, subscriptionEquivUsd, localSessionCount, localEstimatedUsd? }`.&#xA;- Reuse the perf-safe path from `ddx-0a33bc5f` (sessions-index aggregation, shard-aware date filter).&#xA;&#xA;### Part 3 — UI&#xA;&#xA;- Replace the single &#34;Total Cost&#34; card on the Sessions page with three cards:&#xA;  - **Cash paid** ($X.XX) — tooltip &#34;Billed by pay-per-token APIs (OpenRouter, direct API keys)&#34;&#xA;  - **Subscription value** ($X.XX) — tooltip &#34;Dollar-equivalent for tokens consumed under Claude Code / Codex subscriptions. Not cash out of pocket.&#34;&#xA;  - **Local sessions** (N) — tooltip &#34;Sessions served locally. Compute cost not modeled.&#34; If `local_cost_per_1k_tokens` is set, show &#34;$X.XX est.&#34; instead of a count.&#xA;- Per-row: small badge `cash` / `sub` / `local` in the cost column, color-coded (green/blue/gray). Tooltip explains the row&#39;s bucket.&#xA;&#xA;### Part 4 — Optional config&#xA;&#xA;- `.ddx/config.yaml` gains `cost.local_per_1k_tokens: &lt;float&gt;`. When set, local-session total gets rendered as dollars. When unset, count-only. No auto-estimation.&#xA;&#xA;## Out of scope&#xA;&#xA;- Subscription flat-rate cost modeling (&#34;I pay $20/month for Claude Code&#34;). A useful adjacent bead: divide flat rate by consumption to show $/1M-tokens realized value. Not here.&#xA;- Quota trend / headroom. Covered by `ddx-23978824`.&#xA;- Reclassifying historical sessions beyond what reindex can infer from their stored harness/surface fields.

    <acceptance>
**User story:** As a developer looking at the Sessions page, I can tell at a glance how much I actually paid in cash versus what I consumed under subscriptions versus what I ran locally. No number on the page conflates those three categories.&#xA;&#xA;**Acceptance criteria:**&#xA;&#xA;1. **`billingMode` field.** Session records (index rows + live writer) carry `billingMode` ∈ `{paid, subscription, local}`. Enum is validated on write. Unit test covers each value.&#xA;&#xA;2. **Classifier.** `billingModeFor(harness, surface, baseURL)` is the single source of truth. Test table includes at minimum:&#xA;   - claude (Claude Code) → subscription&#xA;   - codex → subscription&#xA;   - openrouter (any model) → paid&#xA;   - openai/anthropic with API key → paid&#xA;   - agent (embedded) → local&#xA;   - openai-compat endpoint pointing at localhost/127.*/192.168.*/10.* → local&#xA;   - openai-compat endpoint pointing at a public host with an API key → paid&#xA;&#xA;3. **Reindex backfill.** Running `ddx agent log reindex` (from `ddx-2ceb02fa`) on a fixture with sessions from all three modes produces index rows with correct `billingMode`. No row is written without one.&#xA;&#xA;4. **Rollup resolver.** GraphQL exposes `sessionsCostSummary(projectId, since?, until?) { cashUsd, subscriptionEquivUsd, localSessionCount, localEstimatedUsd }`. Returns zeros/null correctly when no sessions match. Respects shard-scoped reads from `ddx-0a33bc5f`.&#xA;&#xA;5. **UI — three cards.** The Sessions page replaces the &#34;Total Cost&#34; card with three cards as described (Cash paid, Subscription value, Local sessions). Tooltips explain each bucket. Playwright asserts all three render and that hover on each shows the explanatory text.&#xA;&#xA;6. **UI — per-row badge.** Each session row&#39;s cost column shows a small `cash`/`sub`/`local` badge with a tooltip explaining the bucket. Playwright asserts badge presence and correct classification on a seeded fixture containing one session of each type.&#xA;&#xA;7. **Zero-state.** When the project has zero sessions of a bucket, that card shows `—` / `0`, not `$0.00`, to avoid implying a paid-zero vs absent-data collision. Playwright asserts.&#xA;&#xA;8. **Optional local cost config.** Setting `cost.local_per_1k_tokens` in `.ddx/config.yaml` causes the Local card to render as dollars using the configured rate × sum(totalTokens) for local sessions. Unset: count-only. Integration test covers both paths.&#xA;&#xA;9. **Accessibility.** The three cards and per-row badges expose their bucket meaning via `aria-label`, not just tooltip, so screen-reader users get the same clarification.&#xA;&#xA;10. **Cross-references.**&#xA;    - Depends on `ddx-2ceb02fa` (sessions index) — the `billingMode` field must land in the index schema.&#xA;    - Bead notes in `ddx-23978824` should reference this bead since the unified-endpoints view has overlapping concepts (what&#39;s a paid endpoint vs a subscription harness vs a local endpoint).
