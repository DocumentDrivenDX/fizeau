    <title>Add dedicated read-only tools (ls, find, grep) to ddx-agent for stall detection and efficiency</title>
    <description>
During execute-loop runs with OpenRouter models (GLM-5.1, Qwen 3.6), agents get stuck in read loops — they call read and bash (for ls/find/grep) repeatedly without ever writing. The stall detector cannot distinguish bash ls from bash &#34;git commit&#34; because bash is classified as a write-capable tool, resetting the stall counter on every call.&#xA;&#xA;Adding dedicated read-only tools (ls, find, grep) to ddx-agent would:&#xA;1. Let the stall detector accurately distinguish read-only exploration from write activity&#xA;2. Reduce token usage by avoiding bash prompt/response overhead for simple file queries&#xA;3. Give models clearer affordances — read-only tools for exploration, write/edit/bash for modification&#xA;4. Help models transition from reading to writing faster by making the tool boundary explicit&#xA;&#xA;With these tools, the stall detector classifies read/ls/find/grep as read-only (increment stall counter) and write/edit/bash as write-capable (reset counter). Bash is only used when the agent needs commands that might modify the filesystem.
    </description>
    <acceptance>
dxx-agent has dedicated ls, find, and grep tools; stall detector classifies them as read-only; bash is only classified as write-capable; agents on OpenRouter models transition from reading to writing faster
    </acceptance>
