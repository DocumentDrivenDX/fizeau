    <title>exec: tier-based auto-escalation driven by resource discovery</title>
    <description>
The CLI should not require the user to enumerate harness/model pairs. The tool already knows which tiers exist (model-catalog.yaml: cheap/standard/smart) and which harnesses are configured. Given a live probe of each provider&#39;s health at attempt time, the loop can auto-escalate on its own.&#xA;&#xA;## Principle&#xA;&#xA;Discovery over configuration. The user shouldn&#39;t write a fallback chain — the tool should figure it out.&#xA;&#xA;## Default policy (no flags)&#xA;&#xA;- Start at **cheap** tier, execute the bead&#xA;- If merge succeeds AND post-review (gap #1) APPROVES → close&#xA;- If anything fails (execution_failed, no_changes after N retries, post_run_check_failed, REQUEST_CHANGES from review) → escalate one tier and retry&#xA;- Chain: cheap → standard → smart → exhausted (preserve, flag for human)&#xA;- Review agent runs at max(impl_tier+1, smart). Never review cheap with cheap.&#xA;&#xA;## Resource discovery at attempt time&#xA;&#xA;- Before picking a concrete harness/model for a tier, probe each candidate provider (local inference host, API key validity, network reachability)&#xA;- If provider returns 502 / 500 / timeout / auth-rejected within a short budget, mark it unhealthy with a cooldown and try the next provider in the tier&#xA;- If every provider in a tier is unhealthy, skip the tier and escalate&#xA;- Per-provider health state is process-local and visible via a new &#39;ddx agent providers&#39; sub-command (ddx-09ea1884 Visibility 3)&#xA;&#xA;## Rare overrides&#xA;&#xA;- --min-tier standard (skip cheap — useful for hard beads known to need it)&#xA;- --max-tier standard (never reach smart — cost control)&#xA;- --harness &lt;name&gt; / --model &lt;name&gt; (debug override; disables escalation)&#xA;&#xA;## Surface&#xA;&#xA;Default invocation becomes:&#xA;&#xA;  ddx agent execute-loop --once&#xA;&#xA;Everything else is implicit. Review, escalation, and resource discovery are on-by-default.&#xA;&#xA;## Events&#xA;&#xA;Each attempt appends an execute-bead event: { harness, model, tier, provider, probe_result, attempt_status, review_status }. The event stream shows exactly why a tier/provider was picked or skipped.&#xA;&#xA;## Context&#xA;&#xA;Observed during axon ADR-018 loop:&#xA;- bragi was down (502) — I manually probed with curl and switched to vidar-coder. Tool should do this.&#xA;- qwen3-coder-next merged but opus review rejected — I manually re-invoked with sonnet. Tool should escalate automatically.&#xA;- Resulting CLI invocation in my notes was too complex (—-fallback-chain a/b,c/d,e/f --post-review g/h). User feedback: too many knobs. Reach for a default that just works.&#xA;&#xA;## Acceptance criteria replaced with discovery-driven ones (see accept field)
