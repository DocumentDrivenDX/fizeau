    <description>
Reopening a closed bead after a failing review currently takes 3-4 separate commands: update --status open, update --notes (append), optionally update --description, optionally update --acceptance. This is error-prone and loses the atomicity guarantee that a reopen is a single logical event.&#xA;&#xA;## Scope&#xA;- ddx bead reopen &lt;id&gt; [--reason &lt;text&gt;] [--append-notes &lt;text&gt;]&#xA;- Atomically: set status=open, append a reopen event to the event log, optionally append notes, clear claim fields&#xA;- Emit an event of kind=reopen that shows in ddx bead show&#xA;- Works from CLI and from ddx agent execute-loop review-rejection path&#xA;&#xA;## Context&#xA;Observed during axon ADR-018 loop: first attempt merged, opus review rejected, I had to run three ddx bead update commands plus a description rewrite to get the bead back into a ready-for-second-attempt state. An atomic reopen would have been one command.
