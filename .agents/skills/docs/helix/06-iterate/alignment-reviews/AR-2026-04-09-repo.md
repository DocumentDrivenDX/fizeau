## Intent Summary

- **Vision**: DDx remains the local-first platform where documents are the durable source of truth and the CLI/server/runtime layers make those documents executable.
- **Requirements**: The PRD still centers this repo review on stable CLI/server/website surfaces, reusable tracker and agent infrastructure, process metrics, and a stable plugin API.
- **Features / Stories**: The live repo-wide pressure is now concentrated in FEAT-016, FEAT-018, FEAT-008 frontend governance, FEAT-006 preset/config cleanup, and FEAT-004 tracker hygiene rather than the earlier missing-architecture-frontmatter gaps.

## Implementation Map

- **Topology**: `cli/` contains the Go CLI, tracker, registry, exec, and agent runtime code. `cli/internal/server/frontend/` is the embedded React/Vite SPA. `website/` is the Hugo microsite. `.ddx/beads.jsonl` remains the live execution queue.
- **Entry Points**: `cli/main.go` for the CLI, `ddx server` for HTTP/MCP, `.github/workflows/*.yml` for CI/deploy/security gates, and the published microsite under `website/`.
- **Test Surfaces**: Go command tests under `cli/cmd/`; frontend Playwright under `cli/internal/server/frontend/e2e/`; microsite Playwright under `website/e2e/`.
- **Measured Evidence**: `go test ./cmd -run 'TestAgentExecuteBead|TestAgentRunProfile|TestAgentCapabilities' -count=1` passed from `cli/` on 2026-04-10 UTC, confirming FEAT-006 routing and capability surfaces still work on the current worktree.
- **Unplanned / At-Risk Areas**:
  - `cli/cmd/metric.go` still exposes only the legacy `ddx metric` surface while FEAT-016 requires `ddx metrics` process-metric commands and read-only API endpoints.
  - `cli/internal/registry/registry.go` still relies on `BuiltinRegistry()` as the only package descriptor source, and `cli/cmd/doctor.go` still lacks a `--plugins` audit path.
  - `.github/workflows/ci.yml` now runs frontend Playwright, but it still builds the Go binary before refreshing `cli/internal/server/frontend/dist`, while `cli/internal/server/embed.go` embeds `frontend/dist`.
  - `cli/internal/config/schema/config.schema.json` and `cli/internal/config/types.go` still advertise the obsolete `forge` preset surface, and `cli/internal/agent/agent_runner.go` still accepts configured preset names without validating them.
  - The open tracker still contains 8 open beads without `spec-id`, 46 open beads without `<context-digest>` coverage, and multiple placeholder rows with no durable description according to `ddx bead export --stdout`.
